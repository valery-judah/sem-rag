# Marker Migration Plan

**Goal:** Integrate the custom `marker-master` processing logic (e.g., `CustomFigure`, `SamplePromptProcessor`, `FigureCodeMergeProcessor`) while preserving the production-ready, isolated subprocess architecture of `MarkerRunner`.

## 1. Update Tool Isolation Provisioning
Instead of installing the generic `marker-pdf` package from PyPI, we must configure the isolated environment (located at `tools/marker/.venv`) to install the `marker-master` repository as an editable dependency. This satisfies the `INSTRUCTIONS.md` requirements without violating the "Engine isolation" contract established in `AGENTS.md`.

```bash
uv pip install -e /Users/val/projects/rag/marker-master
```

## 2. Implement Single-File Custom Adapter
The provided `scripts/run_targets.py` in `marker-master` is batch-oriented and relies on a `targets.json` file. To integrate gracefully with the existing `MarkerRunner.run(pdf_path, output_dir, ...)` API, we need a single-PDF adapter script.
**Action:** Create `src/docforge/parsers/pdf_hybrid/engines/custom_marker_single.py` that:
- Accepts `marker_single`-like CLI arguments (`target_pdf`, `--output_dir`, `--page_range`, `--output_format`).
- Imports the custom classes from the editable `marker-master` package.
- Programmatically instantiates `CustomPdfConverter` and runs `converter.build_document(pdf_path)`.
- Uses `MarkdownRenderer` and `JSONRenderer` to write the required outputs directly into the target `--output_dir`.

## 3. Refactor `MarkerRunner` (`marker_cli.py`)
Update the `MarkerRunner` class to point to the isolated Python interpreter executing our new adapter script, rather than the native `marker_single` binary.
- **Discovery:** Update `discover()` to locate the isolated Python executable (`tools/marker/.venv/bin/python`) instead of `marker_single`.
- **Execution:** Update `run()` to build and execute the command array: `[python_bin, "src/docforge/parsers/pdf_hybrid/engines/custom_marker_single.py", str(pdf_path), "--output_dir", str(output_dir), ...]`.
- **Version Detection:** Update `get_version()` to run `[python_bin, "-c", "import marker; print(marker.__version__)"]` to accurately fetch the installed editable package version.

## 4. Verify Output Schema Adapter (`marker.py`)
Verify that `docforge.parsers.pdf_hybrid.engines.marker.adapt_marker_output()` correctly parses the JSON schema produced by `marker-master`'s updated `JSONRenderer.model_dump()`.
- Add a new test fixture generated by `custom_marker_single.py` to `tests/parsers/pdf_hybrid/test_marker_cli.py`.
- Guarantee compatibility with any schema modifications introduced by the custom processors (ensuring "children", "blocks", or "pages" mappings continue to work seamlessly).
